# Computational Inequality: The Forced Binary of Classification Systems

## 1. Description of the Inequality

Computational inequality emerges when classification systems impose discrete categories on continuous phenomena, systematically erasing edge cases and boundary states that do not fit neatly into predefined boxes. This project examines how binary taxonomies—whether in content moderation, image classification, or sensing systems—create structural blind spots that render certain harms invisible, certain uncertainties unrepresentable, and certain populations computationally illegible.

The core inequality lies in the mismatch between human experience (continuous, nuanced, context-dependent) and computational classification (discrete, rigid, decontextualized). When a gesture recognition model forces a "half-raised" hand into either "Hand_Up" or "Hand_Down," or when a content taxonomy requires choosing between "IP Horrorization" and "Death Narrative Implication" for hybrid content, the system does not merely simplify reality—it actively produces ignorance about the very phenomena it claims to measure.

## 2. Summary of the Model

My storyboard presents a five-panel narrative tracing how computational inequality propagates through the machine learning pipeline:

**Panel 1 (Data Origins)** demonstrates how platform metadata systems capture surface-level signals (titles, view counts) while missing critical risk indicators ("Vampire," "Terror" keywords) and cultural context, rendering certain harms invisible from data collection.

**Panel 2 (Classification Enforcement)** reveals how five-dimension taxonomies force hybrid content into discrete categories, collapsing multidimensional meaning. Emotional harm and implied value harm remain entirely uncaptured by the classification schema.

**Panel 3 (Model Behavior)** uses interactive visualization to show how binary gesture classification (Hand_Up/Hand_Down) produces "label flipping" at shoulder height—confidence shifts from 55% to 76% for identical postures—demonstrating forced binary logic on continuous physical movement.

**Panel 4 (Intervention Visualization)** presents a threshold governance interface where budget constraints determine data visibility, showing how pre-algorithmic gates shape what patterns emerge downstream.

**Panel 5 (Theory of Change)** proposes moving from forced binaries to continuous spectra, incorporating uncertainty representation, multi-label classification, and context-dependent thresholds.

## 4. Explanation of the Model

The storyboard functions as both argument and evidence. Each panel contains visual artifacts (screenshots, interactive demonstrations, diagrams) paired with explanatory captions that guide the reader through the inequality's emergence. The narrative structure follows the actual computational pipeline: missing signals in data collection, forced choices in taxonomy design, instability at model inference, governance through thresholds, and finally a proposed reimagining of the entire approach.

The interactive elements—particularly the hand position slider and threshold visualization—allow readers to experience the instability firsthand. By manipulating these controls, users can feel how small changes in continuous input produce abrupt categorical jumps, making abstract concepts of "computational inequality" viscerally concrete.

The model argues that addressing computational inequality requires not better training data or more sophisticated algorithms, but fundamental redesign of classification architectures to accommodate uncertainty, hybridity, and context-dependency. This means adding "boundary" categories, enabling multi-label classification, and building confidence calibration that reflects genuine epistemic uncertainty rather than false precision.
